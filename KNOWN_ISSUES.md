# 已知问题与解决方案

## ✅ 已修复问题

### 问题 1：下载的图片不是真实主题图片（已修复）

**原现象**：
- 请求下载"派蒙"50 张图片
- 实际下载 50 张 Picsum 随机图片（风景为主）

**修复内容**（2026-04-08）：
1. 新增 JSON API 请求方式（`/search/acjson`）
2. 改进 HTML 解析逻辑，支持 `window.data` 和 `__INITIAL_STATE__` 数据提取
3. 添加百度图片 URL 特殊解码功能
4. 优化请求头配置，添加 `Referer` 和 AJAX 标识
5. 实现双策略并行搜索（JSON API 优先，HTML 解析备用）

**当前状态**：✅ 功能已恢复正常，无需配置 Cookie 即可获取真实主题图片

### 问题 2：仅支持百度单一图片源（已修复）

**原现象**：
- 项目只能从百度图片搜索获取结果
- 某些关键词在百度搜索结果较少时无法获取足够图片

**修复内容**（2026-04-13，v2.0.0）：
1. 引入策略模式，抽象 `ImageSource` 基类定义统一接口
2. 实现 `SourceRegistry` 注册表管理所有图片源
3. 新增 `UnifiedImageCrawler` 统一调度器，支持顺序和并行两种搜索模式
4. 新增四个图片源实现：百度、必应、搜狗、360
5. GUI 侧边栏新增图片源多选组件
6. 命令行新增 `--sources` 和 `--parallel` 参数
7. 各图片源的 Cookie 均通过环境变量配置

**当前状态**：✅ 支持多源搜索，自动去重，可累计到目标数量

## 🔧 可选优化

### 配置多源 Cookie（可选优化）

虽然无需 Cookie 即可正常使用，但配置 Cookie 可进一步提高各源的成功率：

```bash
# 1. 获取各平台 Cookie
#   - 打开对应图片网站（如 https://image.baidu.com/）
#   - 按 F12 打开开发者工具
#   - 搜索任意关键词
#   - 在 Network 标签找到请求
#   - 复制 Request Headers 中的 Cookie

# 2. 编辑 .env 文件
# 3. 添加 Cookie 配置（按需配置）
BAIDU_COOKIE=BAIDUID=xxx; BDUSS=xxx; STOKEN=xxx; ...
BING_COOKIE=MUID=xxx; SRCHD=xxx; ...
SOGOU_COOKIE=SUID=xxx; SNUID=xxx; ...
SO360_COOKIE=__sid=xxx; ...

# 4. 重新运行
python main.py "派蒙" 50
```

## 🔧 其他问题

### 问题 3：下载速度慢

**原因**：
- 网络连接不稳定
- 图片源服务器响应慢

**解决**：
- 检查网络连接
- 减少并发线程数（修改 MAX_THREADS）

### 问题 4：下载中断

**解决**：
- 程序支持断点续传
- 直接重新运行即可恢复

### 问题 5：部分图片下载失败

**原因**：
- 图片源服务器不可用
- 图片 URL 已失效

**解决**：
- 程序会自动重试（最多 3 次）
- 失败的图片会记录在日志中
