# 已知问题与解决方案

## ⚠️ 当前限制

### 问题：下载的图片不是真实主题图片

**现象**：
- 请求下载"派蒙"50 张图片
- 实际下载 50 张 Picsum 随机图片（风景为主）

**原因**：
1. 百度图片 API 需要 Cookie 认证才能访问
2. 未配置 Cookie 时，API 返回 HTML 错误页面
3. 程序自动降级到 Picsum 占位图片服务

**解决方案**：

### 方案 1: 配置百度 Cookie（推荐）⭐

```bash
# 1. 复制配置文件
cp .env.example .env

# 2. 获取百度 Cookie
#   - 打开 https://image.baidu.com/
#   - 按 F12 打开开发者工具
#   - 搜索任意关键词（如"派蒙"）
#   - 在 Network 标签找到请求
#   - 复制 Request Headers 中的 Cookie

# 3. 编辑 .env 文件
nano .env  # 或使用其他编辑器

# 4. 粘贴 Cookie
BAIDU_COOKIE=BAIDUID=xxx; BDUSS=xxx; STOKEN=xxx; ...

# 5. 重新运行
python main.py "派蒙" 50
```

**效果**：配置 Cookie 后，可获取真实的百度图片搜索结果。

---

### 方案 2: 使用占位图片（当前默认）

如果不配置 Cookie，程序会使用 Picsum 占位图片服务：

```python
# 占位图片特点:
# - 数量正确（如 50 张）
# - 带关键词种子（保证一致性）
# - 但不是真实主题图片
# - 适合功能测试和演示
```

## 🔧 其他问题

### 问题 2: 下载速度慢

**原因**：
- 网络连接不稳定
- 百度服务器响应慢

**解决**：
- 检查网络连接
- 减少并发线程数（修改 MAX_THREADS）

### 问题 3: 下载中断

**解决**：
- 程序支持断点续传
- 直接重新运行即可恢复
